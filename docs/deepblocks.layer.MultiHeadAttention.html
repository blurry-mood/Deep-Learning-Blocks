deepblocks.layer.MultiHeadAttention module¶

This module contains the implementaion of the multi-head attention mechanism from the Transformers paper 2017, as well as a slightly modified version.

Namely:

  • MultiHeadAttention,

  • MultiHeadAttentionV2:

    This is an implementation of the MHA from but with slight changes. 1. The hidden dimension is always the same as the input, 2. The key is the input itself.

Note

  • Check the original paper for more insights: https://arxiv.org/abs/1706.03762

Parameters

  • num_heads (int, Optional) – denotes the number of heads to use. Default is 8.

class deepblocks.layer.MultiHeadAttention.MultiHeadAttentionV2(input_dim: int, num_heads: int = 8)¶

Bases: torch.nn.modules.module.Module

Initializes a modified Multi-Head Attention module, labelled V2.

It has two differences with the vanilla version, namely, the hidden dimension in each head is a constant equalling to input_dim, and the projection matrix used to obtain the key is always the identity matrix, to reduce the redundancy of parameters engendered by the multiplication of two learnable weight matrices: Key & Query.

This verion is more efficient than the vanilla when the number of heads equals 1, while still having the same capacity.

Parameters

  • input_dim (int) – denotes the number of features of inputs in the sequence.

  • num_heads (int, Optional) – denotes the number of heads to use. Default is 1.

Shape:

  • x (torch.Tensor): A tensor with shape [batch_size, sequence_length, input_dim].

Example

>>> from deepblocks.layer import MultiHeadAttentionV2
>>> mha2 = MultiHeadAttentionV2(22, 8)
>>> x = torch.rand(10, 44, 22)
>>> y = mha2(x)  # y.shape = (10, 44, 22)
